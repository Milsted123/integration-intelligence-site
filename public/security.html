Confidentiality & Data

How we handle your documents

Deal documents are sensitive. This page sets out clearly how your materials are handled; who can access them, how AI processing works, and what happens to them after the engagement.

Integration Intelligence is a founder-led business. Every engagement is run by a named lead, and no document is processed through an automated pipeline without operator review. What follows describes the full picture.

01

Documents stay with the engagement lead

Deal materials you share are received and held by the named engagement lead — not uploaded to a shared platform, a central database, or any third-party service beyond what is disclosed below.

Each engagement is treated as a discrete, independent unit. No content, signals, or findings from one deal are carried into another.

02

Named Engagement Leads

Your documents are accessible only to the named engagement lead or leads identified in your NDA. The individuals with access to your materials are named explicitly in the engagement documentation — there is no offshore processing resource and no automated pipeline with unreviewed access.

03

AI processing — Anthropic Claude API

All AI processing uses the Anthropic Claude API — not the Claude.ai consumer interface. This distinction matters: API access is configured with zero data retention, meaning prompts and completions are not stored on Anthropic's infrastructure after each processing call completes.

Anthropic does not train on API inputs under zero-retention configuration. Your deal content is processed and then gone — it does not persist on Anthropic's servers.

The Anthropic API is the only point at which document content leaves the engagement lead's direct control. It is the sole sub-processor involved in the service.

04

Report delivery

Analysis reports are delivered directly to the named client contact via a secure link. They are not shared with any other party.

05

Documentation available on request

Sub-processor documentation — including Anthropic's Data Processing Addendum available at the API tier — can be provided on request as part of your organisation's vendor review process. Our own data handling documentation, which sets out how we receive, store, and delete engagement materials on our side, is available on the same basis.

For any questions about data handling, contact jon@integration-intelligence.com.

06

Data residency

Engagement materials and outputs are held on UK-based infrastructure. We want to be clear about two points where data crosses borders.

AI processing is performed via the Anthropic Claude API, whose servers are based in the United States. Under zero-retention configuration, content is not stored after each call completes — but it does transit US infrastructure during processing.

Working documents may be held on devices that use Apple's iCloud backup service. Apple stores UK account data in Europe.

If your organisation has specific data residency requirements — including restrictions on US processing — please flag this before engagement begins. We would rather know early than discover a conflict later.
